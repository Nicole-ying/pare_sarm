## Instructions

Produce a structured markdown report with these sections:

### 1. Behavior Trend Summary
- At each evaluation milestone, what is the policy doing in state/action space?
- Is the trajectory improving, regressing, or flat?
- Key numbers: mean_length and the most important task-level metrics at final timestep

### 2. Critical Metrics
- List the 3 most important task-level metrics and explain **why** they are important (impact on progress + cross-metric consistency)
- Flag any metric that is moving in the wrong direction
- Include at least one **cross-metric pattern** (ratio/correlation/consistency).

### 3. Reward Component Health
- Which components are active (mean significantly non-zero)?
- Which components are dominant (|mean| > 2× others)?
- Which are inactive/negligible (mean ≈ 0 or std ≈ 0)?
- Is any component's mean value suspicious (e.g., persistently near-zero across evaluations)?
- Check if any component has **high mean but weak alignment** with task progress metrics

### 4. Behavioral Diagnosis (1-2 sentences)
- In plain language, what is the agent's current strategy?
- Is it reward hacking? Stuck in a local optimum? Making genuine progress?
- Include one sentence on **efficiency** using relative terms (high effort/low gain or low effort/high gain).
- Compare observed behavior against the Task Context above. Does the agent's strategy match the intended task goal? Flag any patterns inconsistent with the task description (e.g., unexpected state distributions, inefficient action patterns, surprising metric correlations).

### 5. Constraint Violations Summary
- Summarize which general principles are currently violated (if any), and rank them by urgency
- Use relative language (dominant share, trend, ratio, concentration), not absolute fixed judgments

### 6. Episode Consistency Summary
- Compare early vs late behavior signals and state whether policy style is consistent or drifting
- Explain if drift appears healthy adaptation or unstable policy behavior

### 7. Key Metrics Numbers
Extract numerical metrics from the Evaluation Metrics table:
- mean_length
- All task-level metrics listed in the Evaluation Metrics table

Output ONLY the report. No code. No proposals.
