== Design Principles ==

1. **Return (total, component_dict).** Total is a scalar float. Component dict maps human-readable names to per-step scalar values. Named components are essential — they enable the system to track which reward signals are active, dominant, or dead during training.

2. **Provide shaping gradients.** The agent should feel progress at every step. Include components that create a smooth gradient from initial state to goal state. The terminal state should have the highest total.

3. **Balance magnitudes.** If one component is 100x larger than another, the agent ignores the smaller one. Target: all components within roughly 50x of each other in expected per-step magnitude.

4. **Terminal signals must not dominate.** Ensure the per-step shaping is strong enough relative to any terminal bonus/penalty. If terminal dominates, the agent learns to end episodes quickly rather than solve the task.

5. **Include _outcome for evaluation visibility.** Add an "_outcome" key to the component dict: 1.0 for success, -1.0 for failure, 0.0 otherwise. Compute total BEFORE adding _outcome so it does not affect learning.
